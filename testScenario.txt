1. 직원의 일반 단건 예약
- 직원 A가 예약 가능한 회의실을 조회하고, 사용 가능한 회의실 1개를 선택해 단건 예약한다.
* 검증 항목:
    직원 재직 상태
    권한등급과 회의실 요구 권한등급 비교
    회의실 상태
    기존 예약 충돌 여부
    예약 생성


2. 권한 부족으로 회의실 조회 제외
- 직원 A의 계산 권한등급이 2등급이고, 회의실 요구 권한등급이 3등급인 경우다.\
* 기대 결과:
    해당 회의실이 예약 가능 목록에서 제외된다.

3. 여러 권한 부여 규칙의 우선순위 계산
- 직원 A에게 다음 규칙이 동시에 적용된다.
    소속 기능 권한: 2등급
    소속 단위 권한: 3등급
* 검증 항목:
    규칙 우선순위
    동일 우선순위일 때 처리 기준
    최종 권한등급 계산 결과

4. 권한 부여 규칙 유효기간 종료
- 직원 A에게 높은 권한이 부여되어 있지만 예약일 이전에 권한 규칙이 종료된다.
* 기대 결과:
    현재 조회 시점이 아니라 실제 예약 대상 시점에 유효한 권한 규칙을 기준으로 판단한다.
    예약 시점에 권한이 유효하지 않으면 예약하지 못한다.

5. 회의실 점검 시간과 예약 시간 중복
- 회의실 상태 이력에 다음이 등록되어 있다.
    점검 시간: 10:30~11:30
    예약 요청: 10:00~11:00
* 기대 결과:
    상태 적용 구간이 일부라도 겹치므로 예약 불가
    날짜뿐 아니라 일시 기준으로 비교

6. 회의실 상태 종료 직후 예약
- 회의실 점검이 11시에 종료되고, 예약은 11시부터 시작한다.
* 기대 결과:
    시간 구간을 [시작, 종료)로 정의했다면 예약 가능
    경계값 처리 규칙 검증

7. 동일 회의실 예약 시간 충돌
-   기존 예약:
    10:00~11:00
    신규 예약:
    10:30~11:30

* 기대 결과:
    신규 예약 저장 실패
    충돌한 기존 예약 정보 반환

8. 취소된 예약 시간 재사용
- 10시 예약이 취소된 이후 다른 직원이 동일 회의실과 시간으로 예약한다.
* 기대 결과:
    취소 상태의 예약은 충돌 대상에서 제외
    취소 이력은 유지
    새로운 예약이 정상 생성

9. 반복 예약 전체 성공
직원 A가 매주 월요일 10시부터 11시까지 4주간 반복 예약한다.
기대 결과:
반복 예약 규칙 1건 생성
예약 인스턴스 4건 생성
4건 모두 동일한 반복 예약 규칙 ID 참조
날짜 계산 정확성 검증

10. 반복 예약 일부 회차 충돌
4개 회차 중 두 번째 주에 기존 예약이 존재한다.
기대 결과:
1·3·4주차 예약 생성
2주차 예약 미생성
성공 3건, 실패 1건과 실패 사유 반환
반복 규칙과 실제 예약 건수 차이 처리

11. 반복 예약 전체 실패
4개 회차가 모두 기존 예약 또는 회의실 상태와 충돌한다.
검증할 정책:
반복 예약 규칙도 저장하지 않는가
규칙은 저장하고 예약 0건 상태를 허용하는가
전체 실패 결과를 어떻게 사용자에게 알리는가
권장 기대 결과:
예약이 한 건도 생성되지 않고 반복 규칙도 롤백하거나 실패 상태로 저장


12. 반복 예약의 남은 회차 일괄 변경
4주 예약 중 두 회차가 지나고, 남은 두 회차의 시간을 10시에서 11시로 변경한다.
기대 결과:
과거 예약은 변경하지 않음
미래 예약 두 건만 조회
변경 전 충돌과 상태 재검사
변경된 예약마다 이력 생성


13. 반복 예약 변경 중 일부 회차 충돌

남은 두 예약을 11시로 일괄 변경하려 하지만, 네 번째 주 11시에는 기존 예약이 존재한다.
검증할 정책:
세 번째 주만 변경하고 네 번째 주는 유지하는 부분 성공
사용자에게 성공·실패 회차 반환


14. 반복 예약 중 특정 회차만 취소
4주 반복 예약 중 세 번째 주만 공휴일 또는 팀 일정으로 취소한다.
기대 결과:
반복 예약 규칙은 유지
세 번째 예약만 취소 상태로 변경
나머지 회차는 유지
취소 내용은 예약 이력에 기록

15. 직원 소속 변경 후 미래 예약 재검증

직원 A가 데이터아키텍처팀에서 다른 조직으로 이동하여 권한등급이 낮아진다.
기존에 높은 권한으로 예약한 미래 예약이 존재한다.

검증 항목:
소속 변경에 따른 권한 실시간 재계산
기존 미래 예약의 유효성 재검증
예약 유지, 자동 취소, 관리자 검토 중 정책 적용
처리 결과와 사유 이력 기록
