## 관계 정의
1. 회사 : 건물 = 1 : 0..N
2. 건물 : 회의실 = 1 : N
3. 부서 : 부서 = 1 : N 
    > 부서는 계층형으로 존재한다.
4. 부서 : 직원 = 1 : N
5. 직급 : 직원 = 1 : N
6. 권한 : 직원 = 1 : N
7. 권한 : 회의실 = 1 : N
8. 직원 : 회의실  = 1 : N
9. 회의실 : 장비 = 1 : N
10. 회의실 : 예약 = 1 : N
    - 회의실 하나는 여러 예약을 가질 수 있지만, 같은 회의실, 겹치는 시간 중복은 안된다.
11. 직원 : 예약(예약자) = 1 : N 
    - 예약자는 한 명만 존재한다.
12. 예약 : 직원(대리자) = M : N
    - 예약 하나에 대리자는 여러 명 존재할 수 있고, 대리자는 여러 예약에 지명될 수 있다.
13. 예약  : 직원(참석자) = M : N
    - 대리자와 마찬가지
14. 직원 : 반복 예약 규칙 = 1 : N
    - 직원은 다수의 규칙을 생성할 수 있다.
15. 예약 : 예약 이력 = 1 : N
16. 예약 : 알림 발송 이력 = 1 : N
17. 예약 : 노쇼 신고 = 1 : 0..1
18. 장비 : 장비 이력 = 1 : N

## 엔터티 속성 정의
1. 직원 : 사번(PK), 이름, 주민번호, 메일, 전화번호, 거주지, 부서, 직급, 권한, 회의실관리자여부
2. 부서 : 부서ID(PK), 부서명, 상위부서ID
3. 회의실 : 회의실ID(PK), 회의실명, 층수, 수용인원, 상태코드(사용가능/점검중/폐쇄), 건물, 관리자사번, 권한
4. 회사 : 회사ID(PK), 회사명, 국가, 지역, 상세주소
5. 건물 : 건물ID(PK), 건물명, 총층수, 회사ID
6. 장비 : 비품ID(PK), 비품명, 종류, 상태, 현재회의실ID
7. 직급 : 직급ID(PK), 직급명
8. 권한 : 권한ID(PK), 권한명, 권한레벨
10. 예약 : 예약ID(PK), 회의실ID, 예약자사번, 시작일시(UTC), 종료일시(UTC), 예약사유, 상태코드, 반복규칙ID, 생성일시
11. 예약 이력 : 예약ID(PK), 회의실ID(FK), 예약자사번(FK), 시작일시(UTC), 종료일시(UTC), 예약사유, 상태코드, 반복규칙ID, 생성일시
13. 반복 예약 규칙 : 규칙ID(PK), 생성자사번(FK), 반복유형(주별/월별), 시작일자, 종료일자, 간격, 반복요일(주별일 때), 반복일자(월별일 때)
14. 대리인 : 예약ID+사번(복합 PK), 등록일시
15. 참석자 : 예약ID+사번(복합 PK)
16. 알림 발송 이력 : 알림ID(PK), 예약ID(FK), 수신자사번(FK), 알림유형코드(확정/30분전/10분전/변경/취소/노쇼확인), 발송일시, 발송상태
17. 노쇼 신고 : 신고ID(PK), 예약ID(FK), 신고자사번(FK), 신고일시, 만료일시(신고+10분), 처리결과(참석확인/자동취소)
